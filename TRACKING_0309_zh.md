## 2. 學習優先級

根據錯題的頻率，以下是學習重點的優先級排序（1 = 高優先級，3 = 低優先級），並完整條列原因：

| **領域**                                   | **優先級**      | **原因**                                                                 |
|--------------------------------------------|-----------------|--------------------------------------------------------------------------|
| 領域 5：安全運營與管理                     | 1 (高)          | - 錯題數量最多<br>- 涵蓋災難恢復 (DRP)、業務連續性 (BCP)、備份策略等多個主題<br>- 涉及物理安全（如滅火器放置）和人員安全（如職責分離）<br>- 安全政策制定和數據銷毀也需加強 |
| 領域 6：安全評估與測試                     | 1 (高)          | - 多個錯題集中在滲透測試目標設定<br>- 審計日誌管理和漏洞評估理解不足<br>- 銀行應用測試等實務知識薄弱 |
| 領域 1：訪問控制                           | 2 (中)          | - RBAC 優勢和單點登錄認證有誤<br>- 安全模型（如 Bell-LaPadula）理解不深<br>- 數據保管人職責和內容/上下文控制需複習 |
| 領域 7：系統與應用安全                     | 2 (中)          | - IDS 異常檢測知識不足<br>- 數據庫安全（如多重更新威脅）有漏洞<br>- 網絡應用開發安全方法需加強 |
| 領域 4：網絡與通信安全                     | 3 (低-中)       | - IPsec 特性理解不全<br>- RFC 1918 私有地址範圍記憶不清<br>- UTP 電纜類別應用不熟練 |
| 領域 3：風險識別、監控與分析               | 3 (低-中)       | - 哈希算法特性不熟<br>- PKI 和 CMM 概念模糊<br>- 威脅建模區分威脅主體與事件有誤 |
| 領域 2：安全運營與管理                     | 3 (低)          | - 緩衝區溢出攻擊理解尚可但不完整<br>- 競爭條件和邊界錯誤需基礎複習 |

### 📌 推薦學習順序
1. **高優先級**：安全運營與管理（領域 5）和安全評估與測試（領域 6）  
2. **中優先級**：訪問控制（領域 1）和系統與應用安全（領域 7）  
3. **低優先級**：網絡與通信安全（領域 4）、風險識別與監控（領域 3）和安全運營（領域 2）

## 3. 學習資源

### 📚 文章與學習指南
- **ISC2 SSCP 學習指南（官方）**  
  - 適用於：所有領域  
  - 資源：ISC2 官網提供電子版或實體書，涵蓋全面考試大綱。  
- **NIST SP 800-53 安全控制**  
  - 適用於：領域 5 (DRP/BCP)、領域 6 (日誌管理)  
  - 資源：NIST 官網免費下載 PDF，搜索 "SP 800-53"。  
- **OWASP Top 10**  
  - 適用於：領域 7 (網絡應用安全)  
  - 資源：OWASP 官網提供中文版，重點閱讀注入和認證相關內容。  
- **SANS 安全意識博客**  
  - 適用於：領域 3 (威脅建模)、領域 5 (人員安全)  
  - 資源：SANS 網站，搜索 "Security Awareness Blog"，有實用案例。

## 最終步驟
1. **優先學習高優先級主題**：專注於領域 5 和 6，以解決大部分薄弱環節。  
2. **利用資源**：觀看推薦視頻並閱讀文章以鞏固概念。  
3. **多做練習**：針對薄弱領域進行專項練習測試，追蹤進步。
